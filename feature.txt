CORE FEATURES (REQUIRED)
================================================================================

1. AUTHENTICATION & USER MANAGEMENT
   ✓ Email + Password authentication via Supabase
   ✓ Patient self-registration (Email + Password). Doctor and Hospital accounts are system-managed (not self-registered).
   ✓ User login/logout functionality
   ✓ User profile management
   ✓ Session management and persistence
   ✓ Password-based authentication only (no social login, OTP, or magic links)

2. ROLE-BASED ACCESS CONTROL
   ✓ Three user roles: Patient, Doctor, Hospital
   ✓ Role-specific dashboards and navigation
   ✓ Server-side role verification
   ✓ Middleware-based route protection
   ✓ Application-level access control for demo stability

3. PATIENT HEALTH DATA MANAGEMENT
   ✓ Patient profile creation and management
   ✓ Health records CRUD operations
   ✓ Vital signs tracking:
     - Blood Pressure (Systolic/Diastolic)
     - Heart Rate
     - Sugar Level
   ✓ Time-series data storage for trend analysis
   ✓ Recorded timestamps for all health data

4. HEALTH DATA VISUALIZATION
   ✓ Interactive charts using Recharts
   ✓ Multi-vital trend visualization over time
   ✓ Line charts for:
     - Blood Pressure trends
     - Heart Rate trends
     - Sugar Level trends
   ✓ Health status indicators
   ✓ Responsive chart design

5. AI-POWERED HEALTH SUMMARIES (Google Cloud Gemini)
   ✓ Integration with Google Cloud Gemini API
   ✓ AI-generated health trend analysis
   ✓ Non-diagnostic explanations in simple language
   ✓ Automatic summary generation from vital signs
   ✓ Server-side API route for secure API key handling
   ✓ Summary display with appropriate disclaimers

6. PATIENT TRANSFER SYSTEM (CORE FEATURE)
   ✓ Transfer request creation ONLY by hospitals
   ✓ Transfer type selection (Emergency/Non-Emergency)
   ✓ Transfer status tracking:
     - Requested
     - Accepted
     - Completed (optional)
   ✓ Transfer acceptance handled ONLY by hospitals
   ✓ Transfer request details and reason
   ✓ Timestamp tracking (requested, accepted, completed)
   ✓ Access control: Patient data only accessible after transfer acceptance
   ✓ Transfer history view for all roles (read-only for doctors and patients)
   ✓ Transfer detail page with full information

7. MEDICAL REPORTS UPLOAD & MANAGEMENT
   ✓ File upload to Supabase Storage (private bucket)
   ✓ Report metadata storage in database
   ✓ Report name and type tracking
   ✓ Secure file URL management
   ✓ Report listing and viewing
   ✓ File download/view functionality
   ✓ Upload timestamp tracking
   ✓ Storage bucket policies for secure access

8. REAL-TIME CHAT (MINIMAL TELEMEDICINE)
   ✓ Supabase Realtime integration
   ✓ Transfer coordination chat
   ✓ Real-time message updates
   ✓ Message history per transfer
   ✓ Sender identification
   ✓ Timestamp display for messages
   ✓ Chat interface for transfer participants
